## Critical Conventions

### dx_app Conventions
1. **Factory pattern** — all Python apps implement IFactory (5 methods)
2. **Absolute imports** — `from dx_app.src.python_example.common.xyz import ...`
3. **CLI args** — use `parse_common_args()` for all entry points
4. **Model resolution** — query `config/model_registry.json`
5. **No hardcoded .dxnn paths** — all from CLI args or registry

### dx_stream Conventions
1. **preprocess-id matching** — DxPreprocess and DxInfer must share the same ID
2. **Queue elements** — place `queue` between every processing stage
3. **DxRate for RTSP** — always rate-limit RTSP sources before inference
4. **Absolute model-path** — DxInfer `model-path` must be absolute
5. **DxMsgConv before DxMsgBroker** — always serialize metadata before publishing

### Universal Conventions
1. **Logging** — use `logging.getLogger(__name__)`, never bare `print()`
2. **NPU check** — `dxrt-cli -s` before any NPU-dependent operation
3. **Headless check** — verify DISPLAY before `cv2.imshow()`
