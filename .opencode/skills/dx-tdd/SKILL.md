# DX Test-Driven Development

Read the full skill document: `.deepx/skills/dx-tdd.md`

## Quick Reference

```
NO APPLICATION CODE WITHOUT A VALIDATION CHECK FIRST
```

1. Define validation checks before creating each file
2. Create minimal code to pass checks
3. Run checks immediately after each file
4. Never batch validation to the end
